GitHub使用指南

2023/12/17 技术

GitHub 是一个基于 Git 的代码托管平台，支持版本控制和协作开发。以下是详细的 GitHub 使用指南，涵盖从基础到进阶的核心功能：

一.GitHub密钥生成与链接

（一）前言

很多朋友在用github管理项目的时候，都是直接使用https url克隆到本地，当然也有有些人使用 SSH url 克隆到本地。然而，为什么绝大多数人会使用https url克隆呢？

这是因为，使用https url克隆对初学者来说会比较方便，复制https url 然后到 git Bash 里面直接用clone命令克隆到本地就好了。而使用 SSH url 克隆却需要在克隆之前先配置和添加好 SSH key 。

因此，如果你想要使用 SSH url 克隆的话，你必须是这个项目的拥有者。否则你是无法添加 SSH key 的。

(二）https 和 SSH 的区别：

1、前者可以随意克隆github上的项目，而不管是谁的；而后者则是你必须是你要克隆的项目的拥有者或管理员，且需要先添加 SSH key ，否则无法克隆。

2、https url 在push的时候是需要验证用户名和密码的；而 SSH 在push的时候，是不需要输入用户名的，如果配置SSH key的时候设置了密码，则需要输入密码的，否则直接是不需要输入密码的。

（三）在 github 上添加 SSH key 的步骤：

第一步、首先，检查下自己之前有没有已经生成：

在开始菜单中打开git下的git bash（当然，在其他目录下打开git bash也是一样的）：
然后执行

ls -al ~/.ssh

第二步、如果能进入到.ssh文件目录下 ，则证明，之前生成过.ssh秘钥，可以直接使用里面的秘钥。

如果不能进入到.ssh文件目录下，则：

检测下自己之前有没有配置：

git config user.name和git config user.email（直接分别输入这两个命令）

如果之前没有创建，则执行以下命令：

注意要先进入到仓库里。

git config --global user.name ‘xxxxx’ 
git config --global user.email ‘xxx@xx.xxx’

生成秘钥

ssh-keygen -t rsa -C ‘上面的邮箱’

代码参数含义：

-t 指定密钥类型，默认是 rsa ，可以省略。
-C 设置注释文字，比如邮箱。
-f 指定密钥文件存储文件名。

接着按3个回车

最后在.ssh目录下(C盘用户文件夹下)得到了两个文件：id_rsa（私有秘钥）和id_rsa.pub（公有密钥）

第三步、如果想登陆远端，则需要将rsa.pub里的秘钥添加到远端。

首先，去.ssh目录下找到id_rsa.pub这个文件夹打开复制全部内容。

接着：

1.登录GitHub，进入你的Settings

2.会看到左边这些目录，点击SSH and GPG keys

3.创建New SSH key

4.粘贴你的密钥到你key输入框中

5.点击Add SSH key

6.再弹出窗口，输入你的GitHub密码，点击确认按钮。

7.到此，就大功告成了。

第四步 测试。

再使用下面的命令测试是否成功

ssh -T git@github.com

二、仓库（Repository）管理

  1. 创建仓库

    • 在线创建：点击 GitHub 主页右上角 + → New repository，填写名称和描述。

    • 本地初始化：

      mkdir my-project
cd my-project
git init
git remote add origin git@github.com:username/repo-name.git

  2. 克隆仓库

    git clone git@github.com:username/repo-name.git

三、基础 Git 操作

  1. 提交代码

    • 添加文件到暂存区：

      git add .         # 添加所有文件
git add file.txt  # 添加特定文件

    • 提交到本地仓库：

    git commit -m “提交描述”

    
- 推送到远程仓库：

    git push origin main # 推送到 main 分支

  2. 同步更新

    git pull origin main  # 拉取远程最新代码

四、分支管理

  1. 创建与切换分支

    git branch feature-1    # 创建分支
git checkout feature-1  # 切换分支
# 或简写：
git checkout -b feature-1

  2. 合并分支

    • 切换到目标分支（如 main），执行合并：

    git merge feature-1

    
- **解决冲突**：手动编辑标记冲突的文件，然后重新提交。

  3. 删除分支

git branch -d feature-1 # 删除本地分支
git push origin –delete feature-1 # 删除远程分支


------

### **五、协作与 Pull Request**

1. **Fork 仓库**
   - 在 GitHub 项目页点击 `Fork`，将他人仓库复制到自己的账号下。
2. **发起 Pull Request（PR）**
   - 修改 Fork 后的仓库代码，推送更改到新分支。
   - 进入原仓库页面，点击 `New Pull Request`，选择分支并提交。
3. **代码审查与合并**
   - 审查者可在 PR 页面评论或请求修改。
   - 通过后点击 `Merge pull request` 合并代码。

------

### **六、Issue 与项目管理**

1. **创建 Issue**
   - 在仓库的 `Issues` 标签页点击 `New Issue`，描述问题或任务。
2. **标签与里程碑**
   - 使用标签（Labels）分类 Issue（如 `bug`、`enhancement`）。
   - 设置里程碑（Milestones）跟踪项目进度。

------

### **七、高级功能**

1. **GitHub Actions**
   - 自动化 CI/CD 流程，配置文件路径：`.github/workflows/xxx.yml`。
2. **GitHub Pages**
   - 托管静态网站：在仓库设置中启用，默认分支为 `gh-pages` 或 `docs/` 目录。
3. **Wiki**
   - 用于编写项目文档，支持 Markdown 格式。

------

### **八、常见问题**

- **撤销本地提交**：

git reset –soft HEAD1 # 保留修改
git reset –hard HEAD1 # 丢弃修改


- **同步 Fork 的仓库**：

git remote add upstream git@github.com:original/repo.git
git fetch upstream
git merge upstream/main


------

### **九、最佳实践**

1. 频繁提交小改动，保持提交信息清晰。
2. 使用 `.gitignore` 排除无需跟踪的文件（如日志、编译产物）。
3. 定期拉取远程更新，避免合并冲突。
