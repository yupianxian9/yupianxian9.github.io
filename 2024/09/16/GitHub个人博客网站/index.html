GitHub个人博客网站

2024/9/16 技术

以下是使用GitHub和Hexo生成个人静态网页的详细步骤，特别针对国内镜像源的配置优化：

一、环境准备与镜像配置

  1. 安装Node.js
    从Node.js中文官网下载安装包，安装完成后验证版本： 

    node -v  # 输出版本号如v18.12.1

  2. 配置npm国内镜像源
    默认npm源速度较慢，建议切换为阿里云镜像： 

    npm config set registry https://registry.npmmirror.com
npm config get registry  # 检查是否生效

  3. 安装Git
    从Git官网下载并安装。验证安装： 

    git --version  # 输出版本号如git 2.42.0

二、Hexo安装与初始化

  1. 全局安装Hexo
    使用npm安装Hexo命令行工具： 

    npm install -g hexo-cli  # 若速度慢可尝试cnpm
hexo -v  # 验证安装

  2. 创建博客项目
    新建一个博客文件夹并初始化Hexo： 

    hexo init

三、GitHub仓库配置

  1. 创建GitHub Pages仓库
    • 登录GitHub，新建仓库，名称为[你的用户名].github.io（如zhangsan.github.io）。
    • 进入仓库设置页，开启GitHub Pages功能，分支选择main或master。

  2. 配置SSH免密登录
    • 生成SSH密钥并添加到GitHub： 

    ssh-keygen -t rsa -C "你的GitHub邮箱"  # 连续回车默认生成密钥
cat ~/.ssh/id_rsa.pub                 # 复制公钥内容

    • 在GitHub设置页的SSH and GPG keys中添加公钥。

四、Hexo项目部署

  1. 修改部署配置
    打开myblog/_config.yml文件，添加GitHub仓库信息： 

    deploy:
  type: git
  repo: git@github.com:你的用户名/你的用户名.github.io.git  # 使用SSH地址
  branch: main  # 与GitHub仓库分支一致

  2. 安装部署插件 

    npm install hexo-deployer-git --save  # 安装Git部署插件

  3. 生成并部署网站 

    hexo clean    # 清理缓存
hexo generate # 生成静态文件（可简写为hexo g）
hexo deploy   # 部署到GitHub（可简写为hexo d）

    • 访问https://你的用户名.github.io查看部署结果。

五、本地调试与自定义

  1. 本地预览 

    hexo server  # 启动本地服务，默认访问http://localhost:4000

  2. 更换主题（可选）
    • 下载主题（如NexT）： 

    git clone https://github.com/theme-next/hexo-theme-next themes/next

    • 修改_config.yml中的theme: next。

注意事项

• 镜像源问题：若npm安装插件失败，可尝试临时切换镜像： 

npm install --registry=https://registry.npmmirror.com

• 配置文件格式：YAML文件缩进需使用空格，冒号后需加空格。
• 自定义域名：在source目录下创建CNAME文件，写入域名（如example.com）并部署。

通过以上步骤，你可以快速搭建一个基于Hexo和GitHub Pages的静态博客，并利用国内镜像源优化安装速度。如需进一步美化或功能扩展，可参考Hexo官方文档或社区主题库。

六、Hexo 主要指令

指令 说明
hexo init <folder> 初始化 Hexo 博客项目到指定文件夹。
hexo generate 生成静态文件（可简写为 hexo g）。
hexo server 启动本地服务器预览（默认 http://localhost:4000，简写为 hexo s）。
hexo deploy 部署到远程仓库（如 GitHub Pages，简写为 hexo d）。
hexo clean 清除缓存和生成的静态文件。
hexo new <title> 创建新文章（默认在 source/_posts 目录下生成 .md 文件）。
hexo new page <name> 创建新页面（例如关于页、分类页）。

(一)创建分类

  1. 确保分类功能已启用

Hexo 默认支持分类功能，但需要确保您的配置文件中没有禁用相关设置。

在 _config.yml 文件中，检查以下配置项：

# 分类 & 标签
default_category: uncategorized
category_map:
tag_map:

确保 default_category 和 category_map 没有被禁用或错误配置。

  1. 创建分类页面

Hexo 需要一个专门的页面来展示所有分类。您可以通过以下命令生成分类页面：

hexo new page categories

这会在 source/categories/index.md 中生成一个文件。

编辑 source/categories/index.md，确保其内容如下：

---
title: 分类
date: 2023-10-01
type: "categories"
---
  • type: "categories" 是必须的，它告诉 Hexo 这是一个分类页面。
  1. 在文章中添加分类

在您的文章 Markdown 文件中，可以通过 categories 字段为文章添加分类。例如：

---
title: 我的第一篇文章
date: 2023-10-01
categories:
  - 教程
  - Hexo
---

一篇文章可以属于多个分类，但通常建议一篇文章只属于一个分类。

（二）创建标签

  1. 确保标签功能已启用

Hexo 默认支持标签功能，但需要确保您的配置文件中没有禁用相关设置。

在 _config.yml 文件中，检查以下配置项：

# 分类 & 标签
default_category: uncategorized
category_map:
tag_map:

确保 tag_map 没有被禁用或错误配置。

  1. 创建标签页面

Hexo 需要一个专门的页面来展示所有标签。您可以通过以下命令生成标签页面：

hexo new page tags

这会在 source/tags/index.md 中生成一个文件。

编辑 source/tags/index.md，确保其内容如下：

---
title: 标签
date: 2023-10-01
type: "tags"
---
  • type: "tags" 是必须的，它告诉 Hexo 这是一个标签页面。
  1. 在文章中添加标签

在您的文章 Markdown 文件中，可以通过 tags 字段为文章添加标签。例如：

markdown

复制

---
title: 我的第一篇文章
date: 2023-10-01
tags:
  - Hexo
  - 教程
---

一篇文章可以添加多个标签。

  1. 生成并预览

运行以下命令生成静态文件并启动本地服务器：

hexo clean && hexo generate && hexo server

访问 http://localhost:4000/tags/，您应该可以看到所有标签的页面。

  1. 在主题中启用标签显示

如果您使用的主题支持标签功能，通常可以在主题的配置文件中启用标签显示。例如，在 _config.[theme].yml 中查找类似以下配置：

# 标签设置
tag:
  enable: true
  per_page: 10

确保 enable 设置为 true。

七、markdown基本语法

（一）基础语法

1. 标题

用 # 表示标题级别，数量对应标题等级（1~6 级）：

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

2. 文本格式

  • 粗体：**粗体文字** 或 __粗体文字__
  • 斜体：*斜体文字* 或 _斜体文字_
  • 删除线：~~删除线文字~~
  • 高亮（部分工具支持）：==高亮文字==
  • 行内代码：代码
  • 换行：行尾添加两个空格或空一行。

3. 列表

  • 无序列表：用 -、+ 或 * 开头：

    - 项目1
- 项目2
  - 子项目（缩进 2 空格）

  • 有序列表：用数字 . 开头：

    1. 第一项
2. 第二项
   1. 子项（缩进 2 空格）

4. 链接与图片

  • 链接：
    [显示文本](链接地址 "标题（可选）")
    示例：[Google](https://google.com "搜索")

  • 图片：
    ![图片描述](图片链接 "标题（可选）")
    示例：![Logo](https://example.com/logo.png "网站图标")

  • 引用式链接（适用于重复使用）：

    [链接文本][id]
[id]: https://example.com "标题"


#### 5. **代码块**

- **行内代码**：``代码``

- **多行代码**：用三个反引号包裹，可指定语言（语法高亮）：

def hello():
    print("Hello World")


#### 6. **引用**

用 `>` 表示引用内容：

```
> 这是引用内容  
> 多行时每行前加 `>`
> 
> 嵌套引用：  
> > 二级引用
```

#### 7. **分割线**

用三个及以上 `-`、`*` 或 `_`（可带空格）：

```
---
***
___
```

### （二）扩展语法（部分工具支持）

#### 1. **表格**

用 `|` 分隔列，`-` 分隔表头与内容，对齐用 `:`：

```
| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :------: | -----: |
| 内容1  |   内容2  |   内容3 |
| 内容4  |   内容5  |   内容6 |
```

#### 2. **任务列表**

用 `- [ ]` 或 `- [x]` 表示未完成/已完成：

```
- [x] 任务1
- [ ] 任务2
```

#### 3. **脚注**

用 `[^标签]` 添加脚注，底部用 `[^标签]: 说明` 定义：

```
这是一句话[^1]。
[^1]: 这是脚注说明。
```

#### 4. **目录（部分工具支持）**

自动生成目录（依赖工具或插件）：

```
[TOC]
```

#### 5. **HTML 嵌入**

直接使用 HTML 标签增强效果（如调整图片尺寸）：

```
<img src="https://example.com/image.jpg" alt="图片" width="200" />
```



运行 HTML

#### 6. **数学公式（需支持 LaTeX）**

- **行内公式**：`$公式$`
示例：`$E = mc^2$`

- **块级公式**：`$$公式$$`

```
$$
\sum_{i=1}^n i = \frac{n(n+1)}{2}
$$
```

#### 7. **流程图 & 时序图（需 Mermaid 等插件）**

使用特定语法生成图表：

graph LR
  A[开始] --> B(结束)

### （三）Hexo 相关语法

#### 1. **Front-matter**

文章开头的 YAML 配置区域，用于设置标题、日期、分类等：

```
---
title: 文章标题
date: 2024-01-01
categories: [教程, Hexo]
tags: [Markdown, 写作]
cover: https://example.com/cover.jpg
---
```

#### 2. **文章摘要截断**

插入 `<!-- more -->` 标签，控制列表页的预览内容：

```
这是预览部分...
<!-- more -->
这是全文剩余内容。
```

#### 3. **资源链接（Hexo 本地文件）**

使用 `{% asset_path %}` 或 `{% asset_img %}` 引用本地资源：

```
![图片描述]({% asset_img example.jpg %})
```
